# AdCraft Studio 产品白皮书


## 一、执行摘要

AdCraft Studio 是一款面向海外金融产品营销场景的 AI 广告视频生产平台，围绕“需求理解—创意编排—脚本生成—分镜组织—导出交付—合规把控”构建端到端创作闭环。产品聚焦数字钱包、理财平台、交易平台、保险产品等高合规要求行业，帮助营销团队在保证内容专业度与监管友好度的前提下，大幅提升广告创意生产效率。

相较于传统视频创作流程依赖多角色协同、长周期沟通和高密度人工修改，AdCraft Studio 通过对话式需求收集、结构化脚本生成、固定分镜生产流程和导出前合规校验机制，将复杂创意流程转化为标准化、可复用、可审查的数字化工作流。

AdCraft Studio 不只是一个视频编辑工具，更是一套为金融营销团队设计的创意基础设施：它将创意策略、脚本框架、视觉设定、合规规则和交付流程统一沉淀在同一个工作台中，使创意资产能够被管理、被复用、被追踪、被规模化生产。

## 二、行业背景与市场机会

### 2.1 海外金融营销正在进入高频视频化阶段

随着数字钱包、移动支付、在线理财、交易平台和保险产品在东南亚、中东、南亚、拉美等新兴市场快速渗透，金融机构和增长团队对短视频广告的依赖不断增强。用户教育、品牌建立、转化拉新、功能说明、活动推广，越来越多地依靠短视频完成。

然而，金融行业天然具有“高专业门槛 + 高表达风险 + 高本地化要求”的特征。一个可投放的视频广告，往往不仅要解决创意问题，还必须兼顾：

- 不同国家和地区的用户理解方式
- 不同产品类型的卖点表达逻辑
- 不同平台的时长、画幅和节奏要求
- 对收益承诺、风险提示、目标受众的合规限制

这使得传统通用型视频创作工具难以真正满足金融产品营销团队的日常工作需求。

### 2.2 金融广告创意生产的核心矛盾

当前大多数金融广告视频生产流程仍存在以下典型问题：

1. 需求输入非结构化，创意反复沟通，前期成本高
2. 文案、分镜、视觉风格由不同角色分散处理，协同效率低
3. 海外市场本地化适配依赖个人经验，复制性差
4. 合规问题常在临近投放或交付阶段才暴露，返工成本高
5. 项目资产沉淀弱，历史案例难复用，创意组织效率低

### 2.3 产品机会

市场需要一款真正为金融营销场景设计的产品，不仅能够“生成内容”，还能够“组织流程、控制质量、降低风险、沉淀资产”。

AdCraft Studio 正是在这一背景下诞生：它以 AI 为生产引擎，以结构化流程为骨架，以合规能力为护栏，以项目管理能力为承载，面向真实的投产型金融营销团队提供可落地的产品能力。

## 三、产品定位

### 3.1 产品定义

AdCraft Studio 是一款服务于海外金融产品营销团队的 AI 广告视频创作平台，提供从需求输入到导出交付的一站式创意生产能力。

### 3.2 产品使命

让金融广告视频的创作从“依赖个人经验的手工协作”，升级为“标准化、结构化、可规模复制的智能生产流程”。

### 3.3 目标客户

AdCraft Studio 面向以下核心客户群：

| 客户类型 | 典型角色 | 主要诉求 |
|----------|----------|----------|
| 金融科技公司 | 增长负责人、品牌营销、内容运营 | 快速生产多支面向不同市场的视频素材 |
| 广告代理与代运营团队 | 客户经理、创意策划、项目运营 | 同时管理多个金融项目，提高交付效率 |
| 海外增长团队 | 区域市场负责人、本地化团队 | 适配不同国家市场的表达与素材组织 |
| 中小创意团队 | 创始人、主理人、自由策划人 | 降低创作门槛，缩短从想法到交付的路径 |

### 3.4 核心价值主张

AdCraft Studio 的核心价值可概括为四点：

- 更快：将视频创意生产流程前置结构化，缩短沟通与制作周期
- 更稳：在脚本与导出阶段内建合规检查机制，减少投放风险
- 更准：围绕金融产品营销场景设计脚本结构、视觉风格和导出流程
- 更可复用：以项目化方式沉淀案例、脚本、分镜和风格配置

## 四、产品能力总览

AdCraft Studio 采用“登录入口 + 项目总览 + 项目工作流”产品结构，形成清晰的工作台体验。

### 4.1 产品入口层

产品通过品牌化登录页面承接首次访问、客户演示和日常使用入口，结合成功案例、业务数据和标准登录能力，建立产品专业感、可信度和商业转化氛围。

### 4.2 项目管理层

项目总览页与项目列表页构成项目管理中台，支持：

- 浏览近期项目
- 查看全部项目
- 按关键词、市场、类型、更新时间筛选
- 从列表页快速进入指定项目阶段
- 统一管理多项目创意生产节奏

### 4.3 创意生产层

产品将创意生产拆解为标准化的五个主要阶段：

1. 需求输入（Brief）
2. 全局设定（Style）
3. 脚本编辑（Script）
4. 分镜脚本（Storyboard）
5. 导出预览（Export）

每个阶段都在统一的数据上下文中承接前一步产出，保证流程连续性和产物一致性。

### 4.4 质量与合规层

在脚本与导出阶段，产品提供面向金融行业的表达风险识别、合规状态展示和 Agent 自动调整建议，帮助业务团队将风险前移到创作流程中解决，而不是在投放临门一脚时返工。

## 五、核心业务流程

AdCraft Studio 的业务流程围绕真实创意交付过程设计，支持从“信息收集”到“投放前复核”的完整闭环。

### 5.1 流程概览

```text
登录进入平台
→ 进入项目总览
→ 新建项目或筛选历史项目
→ 对话式收集广告需求
→ 基于必填信息自动补齐可选信息
→ 自动推荐视觉风格和配乐风格
→ 生成/编辑 AIDA 脚本
→ 固定分镜结构下完成镜头内容编排
→ 启动导出渲染
→ 按状态展示合规检查结果
→ Agent 给出调整建议
→ 用户确认后下载交付
```

### 5.2 流程设计原则

这一流程体现了三项关键设计原则：

- 先收集，再生成：先把信息结构化，降低后续返工
- 先创作，再审查：让创意与合规在同一工作流中衔接
- 先沉淀，再复用：项目与案例沉淀下来，为后续生产提效

## 六、功能模块详解

### 6.1 登录与品牌展示

登录页不是简单的账号入口，而是产品品牌与业务能力的第一展示窗口。页面通过品牌标题、案例封面、成效指标和标准登录表单，对外传递产品定位与业务可信度。

关键能力包括：

- 品牌展示与成功案例呈现
- 登录态管理与退出机制
- 标准化入口体验
- 面向移动端的自适应布局

### 6.2 项目总览与项目列表

项目总览页承载日常入口与最近项目管理；项目列表页则提供完整筛选能力。

支持能力包括：

- 项目统计卡片
- 最近项目卡片浏览
- 全部项目列表浏览
- 按关键词筛选项目
- 按市场筛选项目
- 按产品类型筛选项目
- 按最近更新时间筛选项目
- 快速进入项目对应阶段

这一层能力使 AdCraft Studio 不只适合单个创意人员使用，也适合需要并行管理多个广告项目的团队。

### 6.3 需求输入：对话式 Brief 收集

需求输入页采用对话式交互，而非传统表单录入。系统通过 Agent 与用户持续对话，先收集三项必填信息，再在必要时补齐可选信息。

必填信息包括：

- 产品名称
- 产品类型
- 目标市场

可选信息包括：

- 核心卖点
- 目标受众
- 视频语言

当用户未完整提供可选信息时，系统会在必填信息齐全的前提下进行保守补齐：

- 核心卖点：默认补齐 1-2 条简短、可画面化、易转化的卖点
- 目标受众：根据产品类型与目标市场生成 1 句保守画像
- 视频语言：默认补齐为英语

补齐结果不会静默写入，而会在后续生成前显式展示给用户确认。这种设计既降低了输入门槛，也保证了视频生成前的业务信息质量。

同时，素材上传与自动节点匹配能力让截图、Logo、品牌素材从一开始就进入创作上下文，而不是在后期再零散补充。

### 6.4 全局设定：统一创意基线

全局设定页负责把“抽象创意方向”转化成“可执行风格约束”。

当前可管理的内容包括：

- 品牌 Logo
- 产品名称与 Slogan
- 主色调
- 目标市场
- 视频参数（时长、分辨率）
- 视觉风格
- 背景音乐风格
- 风格预览信息

在该阶段，系统会根据前序需求自动推荐风格结果，用户也可以手动调整。当前视觉风格统一收敛为四类：电影质感、剧情写实、商务场景、产品展示，以便提升输出稳定性和团队协同一致性。配乐风格在这里完成选择，最终配乐在导出阶段统一生成。

### 6.5 脚本编辑：AIDA 结构化创意生成

脚本编辑页以 AIDA（Attention / Interest / Desire / Action）结构为核心，将广告脚本拆为清晰的四段式表达。

该模块支持：

- 文案编辑
- AI 辅助生成
- 策略标签触发
- 动态质量检查
- 风险提示引导
- 自动保存反馈

这种结构化设计特别适合金融营销，因为它能兼顾吸引力、卖点表达、信任建立与行动转化，并天然适配短视频广告时长限制。

### 6.6 分镜脚本：固定结构下的镜头编排

分镜脚本页采用固定分镜结构，不提供任意新增、删除或重构镜头的能力，而是在既定镜头框架中编辑内容细节。这种设计适用于高效率生产场景，可保障节奏稳定、结构清晰和交付一致性。

页面由三部分构成：

- 左侧镜头列表
- 中间主编辑面板
- 最右侧信息栏（镜头帧 + 镜头摘要）

在此基础上，用户可以细化：

- 镜头标题
- 起止时间
- 时长
- 画面描述
- 景别与运镜
- 旁白与字幕
- AI Prompt
- 关联素材

这种设计让分镜阶段既保留结构控制，又保留创意调整空间。

### 6.7 导出预览：渲染与合规联动

导出预览页是创意生产流程的最后一道关口，也是 AdCraft Studio 的核心差异化能力之一。

平台支持：

- 不同平台导出适配（TikTok / Instagram / YouTube）
- 渲染状态展示
- 待渲染 / 渲染中 / 已完成等状态反馈
- 多状态合规检查区（待检查 / 待结果 / 已完成）
- 合规不通过样例展示
- Agent 自动调整建议外显
- 应用调整后放行下载

特别值得强调的是，合规能力并不是“一个结果提示”，而是“一个可交互的调整流程”。用户可以清楚看到：

- 当前合规检查处于什么状态
- 哪段表述存在风险
- Agent 建议如何修改
- 应用修改后状态如何变化
- 合规通过后才能正式下载

这使 AdCraft Studio 在金融行业场景下具备了更强的投产可用性。

## 七、AI 能力体系

AdCraft Studio 的 AI 能力并非孤立功能点，而是嵌入工作流各阶段的系统化能力。

### 7.1 需求理解能力

通过对话方式从自然语言中提取结构化营销需求，降低输入门槛，提高 Brief 完整性。

### 7.2 创意编排能力

通过 AIDA 框架和策略标签，将营销目标转化为符合短视频传播逻辑的脚本结构。

### 7.3 风格推荐能力

根据产品类型、市场和表达目标自动推荐视觉风格和配乐风格，降低创意决策成本。

### 7.4 合规辅助能力

识别绝对化表达、风险提示缺失等问题，并以用户可理解的方式外显自动调整建议。

### 7.5 工作流连接能力

AI 不只是输出内容，还参与阶段衔接：从需求进入全局设定，从脚本进入分镜，从渲染进入合规检查。这种串联能力是平台生产效率提升的关键来源。

## 八、合规体系设计

对于金融营销场景而言，合规不是附属能力，而是产品设计的核心组成部分。

AdCraft Studio 的合规体系遵循以下原则：

### 8.1 风险前移

把风险识别前移到创意生产和导出前，而不是在投放阶段被动发现。

### 8.2 表达可解释

当系统判定不通过时，不只是给“红灯”，还明确指出风险原文，并给出替代建议。

### 8.3 Agent 外显协同

Agent 的调整过程对用户可见，用户理解修改原因后再应用，而不是黑盒替换。

### 8.4 下载前放行控制

在关键风险未处理前，下载行为受到限制，从流程上形成控制闭环。

这种机制使产品既适合高频生产，也适合需要多轮审查和跨角色确认的企业使用场景。
